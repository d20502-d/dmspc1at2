Musical Note UI Prototype — Concept and Progress

Prototype live link:

  • https://dmspc1ui.netlify.app/

Inspirations:

Traditional Musical Instruments (Piano Keyboard):

The spatial layout and octave structure are inspired by the piano keyboard’s organization of notes and octaves. This familiar framework helps users intuitively understand the relationship between notes.

Grid-Based Layout (MIDI Grid Controllers):

I was also inspired by MIDI grid controllers, such as the Novation Launchpad and Ableton Push, which use a matrix of buttons arranged in rows and columns to trigger musical notes and samples. Similarly, my 5x3 button grid maps spatial positions to musical notes, providing a clear and playable visual representation of music.

Multiple Input Methods:

MIDI controllers often support various input methods, including pads, knobs, and buttons. Reflecting this versatility, my project supports mouse, touch, and keyboard inputs, allowing users to interact with the instrument in multiple ways and enhancing accessibility.


Project Conceptualisation

Spatial Control:

The interface uses a clear 5x3 grid layout to represent musical notes, mapping spatial positions directly to piano scales. This spatial arrangement helps users visually and physically understand the relationships between notes and octaves, making the musical experience more intuitive and engaging.

Visual Distinction of Octaves:

By visually distinguishing the first note of each octave with an additional diamond-shaped frame, the design conceptually emphasizes musical structure and hierarchy, aiding user orientation within the note grid.

Novel Input and Interaction:

While maintaining accessibility through traditional input methods (mouse, touch, keyboard), the project introduces a unique keyboard layout and a “Surprise Me” feature that plays random notes using Math.random(). This adds an element of playfulness and exploration, encouraging users to interact with the instrument in unexpected ways.

Customize

The settings panel allows users to toggle the display of note names, keyboard visualization, and a metronome, giving control over the interface’s complexity and aiding different learning or playing styles.


Technical Approach

UI Implementation:

HTML and CSS create the grid layout with semi-transparent squares, circular and diamond frames.

Audio Playback:

Web Audio API plays notes mapped to each button.

Advanced Technique:

Math.random() is used in the "Surprise Me" feature to select and play random notes.

Input Methods:

Supports mouse, touch, and keyboard inputs for accessibility.

Visual Feedback:

Pressed keys scale up and change color briefly to provide immediate feedback.

Settings Toggles:

Sliding toggle buttons control visibility of note names, keyboard display, and metronome activation.
